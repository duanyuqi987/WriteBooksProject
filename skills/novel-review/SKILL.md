# 小说审查

本工序审查当前章节、短篇或片段。目标不是把所有文字改成同一种风格，而是判断它是否服务这部小说自己的目标。

## 启动读取

标准项目必须读取：

- 待审正文或 `章节/chapter-xxx.md`
- `project.md`
- `outline.md`
- `memory-snapshot.md`
- 相关人物卡
- 相关伏笔文件
- `skills/novel-writing/references/ai-flavor-banlist.md`

如果是轻量片段审查，只基于用户提供文本审查，并明确哪些一致性问题无法确认。

## 六层检查

### 1. 叙事有效性

- 这一章/段是否有清晰功能？
- 删掉后故事、人物或读者理解是否会损失？
- 是否发生了可感的变化：事件、关系、信息、情绪或主题理解？

### 2. 人物一致性

- 角色欲望、恐惧、选择是否一致？
- 对话是否能区分人物？
- 角色是否为了作者推进剧情而突然变笨、变聪明或变性格？

### 3. 结构节奏

- 开头是否承接上文或迅速建立问题？
- 中段是否有阻力和变化？
- 结尾是否符合该作品取向，而不是模板化悬念？
- 快慢是否单调，是否有无效过渡？

### 4. 语言质感

- 是否有空泛形容、套话、解释腔、总结腔？
- 是否有过度书面化或过度网文化的问题？
- 关键情绪是否被动作、细节、场面承载？
- 比喻、成语、金句是否抢戏？

### 5. Canon 与伏笔

- 是否违反既有设定？
- 是否凭空加入未记录的重要元素？
- 伏笔是否自然埋入、提示或兑现？
- 反转是否有前置依据？

### 6. 读者体验

- 是否达成 `project.md` 中定义的核心感受？
- 文学向是否有余味，类型向是否有驱动力，悬疑向是否有信息控制，连载向是否有追读动力？
- 是否存在读者容易弃读、跳读或出戏的位置？

## 判定

| 判定 | 条件 | 动作 |
|---|---|---|
| pass | 主要目标达成，仅有轻微问题 | 进入 `novel-update` |
| minor_fix | 方向正确，有 1-3 个局部问题 | 给出最值得改的局部建议 |
| rewrite_required | 核心功能没完成或人物/结构明显错位 | 指出 1-2 个重写目标 |
| reject | 与项目方向或 canon 根本冲突 | 回到大纲或项目设定 |

## 修改意见格式

```markdown
**review_status:** pass / minor_fix / rewrite_required / reject
**review_focus:** 叙事 / 人物 / 语言 / 结构 / canon / 体验

### 结论
[一句话说明是否能进入下一步]

### 主要问题
1. [问题 + 影响 + 建议改法]

### 可保留之处
[说明哪些部分有效，避免误删]

### 下一步
[进入 update / 局部修改 / 重写 / 回到 outline]
```

## 红线

- 不把“更像网文”当作通用评价标准。
- 不把“更文艺”当作通用评价标准。
- 不只说好坏，必须给可执行修改点。
- 不在没有上下文时武断判断 canon。
- 不为了去 AI 味而把所有句子改短、改冷、改碎。
