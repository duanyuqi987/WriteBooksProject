# 通用中文小说写作台

这是一个面向中文小说创作的技能系统。它不预设用户一定要写网文、爽文或长篇连载，而是先确认作品形态、读者体验、叙事目标和写作约束，再选择合适的流程强度。

## 工作原则

1. **目标优先**：先判断用户要解决的是灵感、结构、正文、修改、续写还是记忆管理。
2. **作品适配**：短篇、中篇、长篇、连载、文学向、类型向使用不同规划粒度。
3. **中文表达优先**：正文、分析、修改建议默认使用自然简体中文。
4. **文件为真相源**：已有项目必须先读项目文件，不依赖对话记忆直接续写。
5. **最小有效流程**：简单任务不强行套完整流程，长篇连续创作才启用完整状态机。
6. **小说边界清晰**：不把小说 skill 用于技术书、报告、教程、营销文案等非小说任务。
7. **作者署名固定**：所有新建小说项目、正文合订本、项目底座和发布说明的作者署名默认写为“段锦佑”，除非用户在当前对话中明确要求临时改用其他署名。
8. **书名通俗有吸引力**：起小说名时优先清楚、好懂、有画面、有阅读欲，让读者一眼知道故事入口并愿意点开；避免只追求诗化、晦涩、抽象或拼音化。
9. **保字数精修**：用户要求“优化、精修、打磨”时，默认不是压缩字数，而是在原有体量上提升文字、内容、场景、对话、动作、细节和现场感。除非用户明确要求删减或缩写，否则长篇精修后的总字数应基本保持原目标体量，单轮改动建议控制在原字数的 ±5% 到 ±10% 内；删掉空泛解释后，必须用有效场景、人物行动、对话和生活细节回补，不把长篇改成骨架稿。

## 总流程

```text
用户请求
  ↓
识别任务类型：新作 / 续写 / 大纲 / 正文 / 修改 / 审查 / 记忆更新
  ↓
加载项目文件：project.md、outline.md、project-state.md、memory-snapshot.md、人物/、伏笔/
  ↓
选择流程强度：轻量一次性 / 标准项目 / 长篇连续
  ↓
执行对应工序：brainstorm → outline → draft → review → update
  ↓
写入文件并汇报：改了什么、为什么、当前进度、下一步
```

## 流程强度

| 模式 | 适用场景 | 要求 |
|---|---|---|
| 轻量一次性 | 单个灵感、片段、短篇、单章润色 | 不强制建完整项目文件，但要说明假设 |
| 标准项目 | 中短篇、明确书名和设定的小说 | 建 `project.md`、`outline.md`、人物卡和章节文件 |
| 长篇连续 | 多卷、多章、长期续写 | 启用完整状态机、记忆快照、伏笔追踪和章节状态 |

## 工序路由

| 用户意图 | 路由 |
|---|---|
| 想写一本新小说、只有粗略想法 | `skills/novel-brainstorm` |
| 已有设定，需要大纲、卷纲、章节路线 | `skills/novel-outline` |
| 写正文、续写下一章、改写片段 | `skills/novel-draft` |
| 审稿、查问题、去 AI 味、检查一致性 | `skills/novel-review` |
| 一章通过后同步记忆、角色、伏笔、状态 | `skills/novel-update` |
| 录音转文字、整理口述素材 | `skills/novel-whisper` |

## 项目目录

长篇或持续项目默认使用：

```text
docs/YYYY-MM-DD/小说/<中文书名>/
├─ project.md              # 项目底座：题材、主题、角色、世界、风格、边界
├─ outline.md              # 大纲：结构、章节、状态、滚动规划
├─ project-state.md        # 调度快照：当前章、阻塞点、下一步
├─ memory-snapshot.md      # 近期记忆：最近章节摘要、累计变化
├─ 人物/<角色名>.md        # 角色卡：欲望、矛盾、关系、声纹、变化
├─ 伏笔/foreshadow-xxx.md  # 伏笔追踪：埋入、提示、兑现、状态
├─ 章节/chapter-xxx.md     # 章节工作文件
└─ 正文/                   # 纯正文文件
```

短篇或一次性任务可以只产出正文和简短说明，不强制创建完整目录。

## 交付汇报

每次完成后用中文简明汇报：

1. 当前处理的作品、章节或片段
2. 本轮完成了什么
3. 涉及文件
4. 字数或结构变化
5. 已验证的检查项
6. 尚未解决的问题和下一步
