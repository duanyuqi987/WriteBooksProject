# 双盲审查系统

深度融合 Santa Method 的多智能体对抗验证框架。核心洞察：单个智能体审查自己的输出，共享相同的偏见、知识盲区和系统性错误。两个独立的审查者打破这种失败模式。

## 架构

```
阶段 1：生成者 → 写出章节
阶段 2：双独立审查 → 审查者 B + 审查者 C（同一标准，不共享上下文）
阶段 3：裁决门 → 两者都必须通过。无例外。
阶段 4：修复循环 → 收集所有问题，修复所有问题，重新审查，直到收敛
```

## 阶段 2：双独立审查关键不变量

1. **上下文隔离** — 两个审查者互不知对方的评估
2. **相同评分标准** — 两者收到完全相同的评估准则
3. **相同输入** — 两者收到原始规格（outline 任务 + project 设定）和生成的章节
4. **结构化输出** — 每个审查者返回类型化判定，不是散文

## 审查者提示模板

```
你是一个独立的质量审查者。你没有见过这份输出的任何其他审查结果。

## 章节规格
{outline 任务 + 推荐技法 + 字数目标}

## 项目设定
{project.md 风格 + 世界观规则 + 禁止事项}

## 待审查的章节
{章节正文}

## 评估准则
对以下每一条准则，评估章节是否通过：

### 1. 任务完成度
- PASS：章节完成了任务说明中的所有要点
- FAIL：[具体指出遗漏了什么]

### 2. Canon 一致性
- PASS：没有与 project.md 或前文矛盾
- FAIL：[具体指出矛盾点]

### 3. 技法运用
- PASS：推荐的技法得到了有效运用，没有技法堆砌感
- FAIL：[具体指出技法问题]

### 4. 升级感
- PASS：冲突/信息/处境至少有一项比前一章进步
- FAIL：[指出哪里原地踏步]

### 5. AI 腔
- PASS：没有发现 6 大类 AI 腔问题
- FAIL：[具体指出哪一类，什么地方]

### 6. 章末钩子
- PASS：用动作/发现/选择/危机收尾，没有抒情总结
- FAIL：[具体指出问题]

返回你的评估作为结构化结果：
- verdict: PASS 或 FAIL
- 如果 FAIL，列出 critical_issues（必须修复的具体问题）
- suggestions（建议改进但不是必须的）
```

## 裁决门逻辑

```
def 裁决(审查B, 审查C):
    if 审查B.verdict == "PASS" AND 审查C.verdict == "PASS":
        return "通过"  # 发布
    
    所有问题 = 去重(审查B.critical_issues + 审查C.critical_issues)
    所有建议 = 去重(审查B.suggestions + 审查C.suggestions)
    return "不通过", 所有问题, 所有建议
```

**为什么两者都必须通过**：如果只有一个审查者发现问题，那个问题是真实存在的。另一个审查者的盲点正是双盲审查要消除的。

## 收敛循环

```
最大迭代次数 = 3

for 第N轮 in range(3):
    判定 = 裁决(审查B, 审查C)
    if 判定 == "通过":
        return 发布(章节)
    
    章节 = 修复(章节, 所有问题, "只修复被标记的问题，不要改动其他内容")
    # 用全新智能体重跑审查（不记忆上一轮的结果）
    
if 3轮还不过:
    上报给用户(章节, 剩余问题)
```

## 修复循环规则

1. 只修复被标记的具体问题，不随意改动其他内容
2. 每次修复后两个审查者必须重新审查（全新上下文）
3. 最多 3 轮，防止无限循环
4. 3 轮后仍不通过 → 上报用户做决策

## 指标追踪

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 一次通过率 | >70% | 第一轮就通过的比例 |
| 平均迭代次数 | <1.5 | 达到通过的平均轮数 |
| 问题分类 | - | 什么问题最多（AI腔/Canon/升级感） |
| 审查者一致率 | - | 两个审查者判断一致的比例 |

## 简化模式（单章日常使用）

对于日常章节审查，除非有明显问题，可以用简化模式：

- 只在章节 draft 完成后执行一次双盲审查
- 如果一次通过率 >90%，继续使用简化模式
- 如果出现问题积累（连续 3 章需要 2+ 轮修复），切回完整模式

## 失败模式与缓解

| 失败模式 | 症状 | 缓解 |
|---------|------|------|
| 无限循环 | 审查者不断发现新问题 | 最大迭代次数 3 轮，超标上报 |
| 橡皮图章 | 每次都通过 | 对抗性提示："你的工作是找问题" |
| 主观漂移 | 风格偏好被标记为问题 | 收紧评估标准，只用客观条件 |
| 修复引入新问题 | 修 A 引入 B | 每轮用全新审查者 |
| 审查者一致性偏差 | 两个都漏掉同一个问题 | 增加第三个审查者或人工抽查 |
