# 大纲规划

四层展开系统，将 project.md 中的目标字数和设定转化为可执行的章节路线。

## HARD-GATE

- 未读取 project.md 不得开始规划
- 只详细规划当前卷，后续卷只给方向种子
- 新卷规划必须先读取已完成的章节摘要（memory-snapshot.md），基于实际走向调整

## 四层展开模型

```
第 0 层：字数规模（已在 brainstorm 确认）
  ↓
第 1 层：卷结构 → 每卷任务 + 字数分配 + 起点/终点状态
  ↓
第 2 层：粗略章节 → 每章一句任务说明 + 结构标记
  ↓
第 3 层：精细章节 → 每章场景拆分 + 技法标注 + 伏笔标记
```

## 检查清单

### 首次规划（无 outline.md）

1. **读取 project.md** → 提取 premise、角色、世界观、冲突、风格、字数、技法偏好
2. **第 1 层：卷结构规划** → 确定卷数、每卷字数、每卷起点/终点状态
3. **第 2 层：粗略章节** → 当前卷每章一句任务说明 + 结构标记
4. **第 3 层：精细当前卷前 5 章** → 前 5 章做精细规划（场景拆分 + 技法标注 + 伏笔标记）
5. **伏笔规划** → 确定全书的伏线地图（至少 3 条大伏线）
6. **技法分配** → 为每章标注建议使用的技法
7. **后续方向** → 每卷 1-2 句话种子
8. **交付确认** → 保存 outline.md，创建 `伏笔/` 文件夹

### 新卷规划（当前卷已全部 done）

1. 读取 `memory-snapshot.md` → 获取已完成卷的实际走向
2. 读取 `project.md` → 检查是否有 canon 变更
3. 读取所有未兑现伏笔 → 确保新卷规划中纳入
4. 规划新卷：先粗略（全部章），再精细（前 5 章）
5. 更新 `project-state.md`

## 第 1 层：卷结构格式

```markdown
## 全书规模
- 目标字数：XX 万字
- 每章字数：约 XXXX 字
- 总章数：XX 章
- 卷数：X 卷

## 卷结构

| 卷号 | 章范围 | 章数 | 预估字数 | 本卷任务 | 起点状态 → 终点状态 |
|------|--------|------|---------|---------|-------------------|
| 第1卷 | 1-45 | 45章 | ~10万字 | XXX | XXX → XXX |
| 第2卷 | 46-90 | 45章 | ~10万字 | XXX | XXX → XXX |
```

## 第 2 层：粗略章节格式

```markdown
| 章节 | 任务说明 | 结构标记 | 推荐技法 | 伏笔 | 状态 |
|------|---------|---------|---------|------|------|
| chapter-001 | [一句话任务] | setup | 白描+反衬法 | - | planned |
| chapter-002 | [一句话任务] | build | 一击两鸣 | plant:foreshadow-001 | planned |
```

## 第 3 层：精细章节格式（仅当前卷前 5 章）

```markdown
### chapter-001：[章节标题暂定]

**任务**：[从粗略章节继承]
**结构标记**：setup
**推荐技法**：白描、反衬法

**场景拆分**（3-5 个场景）：
1. [场景一]：[一句话描述]（约 400 字）
2. [场景二]：[一句话描述]（约 600 字）
3. [场景三]：[一句话描述]（约 500 字）
4. [场景四]：[一句话描述]（约 400 字）
5. [场景五]：[钩子]（约 300 字）

**本章目标字数**：2200 字
**本章关键信息**：[本章必须传达的核心信息]
**本章伏笔**：[埋入/提示/兑现的伏笔 ID]
```

## 技法分配原则

根据章节类型自动推荐技法（详见 `skills/novel-writing/references/writing-techniques.md` 技法映射表）：

| 结构标记 | 主推荐技法 | 辅推荐技法 |
|---------|-----------|-----------|
| setup | 白描 + 反衬法 | 一击两鸣 |
| build | 一击两鸣 + 草蛇灰线 | 暗透法 |
| climax | 繁章波折法 + 白描 | 春秋笔法 |
| fallout | 避繁文法 + 不写之写 | 层峦迭翠 |

连续 3 章不使用同一技法组合。

## 伏笔规划

在 outline 阶段确定全书的伏线地图（详见 `skills/novel-writing/references/memory-system.md` 伏笔格式）：

- 至少 3 条大伏线（跨卷）
- 至少 5 条小伏线（跨章）
- 标注每条伏线的埋入章、中途提示章、兑现章

## 滚动展开规则

- 当前卷前 5 章：精细规划（场景拆分 + 技法标注）
- 当前卷第 6-末章：粗略规划（一句任务说明 + 结构标记 + 技法推荐）
- 下一卷：方向种子（1-2 句话）
- 再往后：卷名和大致方向

**写完 5 章后**：在写第 6 章之前，对第 6-10 章做精细展开。以此类推，保持"已精细规划 5 章 ahead"。

## 交付

1. 展示卷结构表
2. 展示当前卷粗略章节列表
3. 展示前 5 章精细规划
4. 展示伏线地图
5. 用户确认后：
   - 保存 outline.md
   - 创建 `伏笔/` 文件夹及伏笔文件
   - 更新 `project-state.md`（状态：planned）
   - 调用 `skills/novel-writing` 路由到 `skills/novel-draft`

## 核心原则

- **字数为锚**：每卷每章的字数分配不能超过总目标
- **四层展开**：不越级规划，粗略→精细逐步细化
- **滚动窗口**：始终只精细规划 5 章 ahead
- **技法前置**：每章标注技法，写作时可直接参考
- **伏笔可追溯**：每条伏笔有独立文件和状态
