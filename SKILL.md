# 小说写作台

纯小说写作系统。不写技术书、不写报告、不写教程。只写小说。

## 核心流程

```
用户输入（含目标字数）
  ↓
步骤 0：字数评估 → 计算卷数、章数
  ↓
步骤 1：检查记忆系统 → 恢复上次进度（文件真相源）
  ↓
步骤 2：路由到对应工序状态机
  ↓
步骤 3：执行工序 → 六层质量闸门（含双盲审查）→ 6项记忆写入
```

## 核心系统

| 系统 | 来源 | 功能 |
|------|------|------|
| 角色工坊 | persona-forge | 身份张力公式 + 底线规则 + 声纹档案 |
| 声纹系统 | brand-voice | 角色声纹 + 叙事者声纹 + 差异化对比 |
| 三遍校对 | huashu-proofreading | 内容 → 6类AI检测 → 细节打磨 |
| 双盲审查 | Santa Method | 双独立审查者 + 裁决门 + 收敛循环 |
| 体验规格 | iPhone 团队 | 人的时刻 → 感受 → 技术约束 |
| 任务管理 | task-harness | JSON 真相源 + 增量式进度 + 5秒恢复 |
| 10种技法 | 红楼梦分析 | 写作技法库 + 场景映射表 |
| 记忆系统 | 自主设计 | 7层记忆：快照 + 伏笔 + 角色 + 大纲 |
| Skill查找 | 自主设计 | prompt → 技法/第三方技能/风格文件匹配 |
| 类型技法映射 | 起点分析 | 8种小说类型 × 技法组合 × 策略 |
| 起点分析器 | 爬虫脚本 | 每次启动获取Top100热门小说分析 |
| Whisper 语音转写 | OpenAI Whisper | 本地语音转文字，音频不出本机 |
| 小说数据集 | HuggingFace/Mirror | 下载中文网文数据集用作写作参考 |

## 步骤 0：字数评估（每次启动必做）

根据用户目标字数，计算写作规模：

| 全书目标字数 | 建议章数 | 建议卷数 | 每章字数 |
|-------------|---------|---------|---------|
| 10 万字（短篇） | 40-50 章 | 1-2 卷 | 2000-2500 |
| 30 万字（中篇） | 120-150 章 | 2-3 卷 | 2000-2500 |
| 50 万字（中长篇） | 200-250 章 | 3-5 卷 | 2000-2500 |
| 80 万字（长篇） | 320-350 章 | 4-6 卷 | 2200-2800 |
| 100 万字以上（超长篇） | 400+ 章 | 6+ 卷 | 2200-3000 |

用户可自定义每章字数。每卷 40-60 章为宜。

## 步骤 1：加载记忆系统（每次启动必做）

必须先检查项目文件是否存在，恢复上次写作进度：

1. 检查 `project.md` → 有则读之
2. 检查 `outline.md` → 有则读之，获取当前进度
3. 检查 `project-state.md` → 有则读之，获取上次状态
4. 检查 `memory-snapshot.md` → 有则读之，获取最近章节摘要
5. 检查 `伏笔/` → 有则读取所有未兑现伏笔
6. 检查 `人物/` → 有则读取所有角色当前状态
7. 检查 `qidian-analysis.md` → 有则读之
8. 必要时运行 `scripts/qidian_analyzer.py` 获取最新热门小说数据
9. 检查 `数据集/` → 有则读之
10. 必要时运行 `scripts/novel_dataset_downloader.py` 下载新数据集

**无文件 → 新项目，进入 brainstorm。有文件 → 从上次中断处继续。**

### 步骤 1.5：Skill 查找（写作前执行）

根据当前写作场景，查询 `skills/novel-writing/references/skill-finder.md`：
- 当前场景类型匹配对应技法
- 当前工序匹配对应第三方技能
- 当前小说类型匹配对应风格文件
- 返回 top 3-5 推荐，写入当前章 planning 备注

## 步骤 2：路由到状态机

| 状态 | 触发条件 | 路由目标 |
|------|---------|---------|
| idea | 无 project.md | `skills/novel-brainstorm` → 字数驱动头脑风暴 |
| planned | 有 project.md，章节 status=planned | `skills/novel-outline` → 大纲分层展开 |
| drafting | 当前章 status=drafting | `skills/novel-draft` → 写正文 |
| reviewing | 当前章 status=reviewing | `skills/novel-review` → 审查 |
| done | 当前卷全部 done | `skills/novel-outline` → 规划下一卷 |
| blocked | project-state.md 有 blocked_reason | 按原因路由 |
| transcribe | 用户说"录音转文字"/"转写"/"字幕" | `skills/novel-whisper` → 语音转文字 |
| dataset | 用户说"下载小说数据"/"爬小说"/"数据集" | `scripts/novel_dataset_downloader.py` → 下载参考素材 |

详细状态机规则见 `skills/novel-writing/references/state-rules.md`。

## 核心原则

1. **先字数为王**：所有规划以目标字数为起点反向推导
2. **分层展开**：字数 → 卷 → 粗略章 → 精细章，不越级规划
3. **记忆驱动**：每次会话从文件恢复，不依赖对话记忆
4. **技法嵌入**：10 种写作技法在写作中灵活运用
5. **持续迭代**：每完成一章立即写入记忆，支持随时中断和恢复

## 文件路径规范

```
docs/YYYY-MM-DD/小说/<书名 slug>/
├─ project.md              # 项目底座
├─ outline.md              # 大纲（逐章状态跟踪）
├─ project-state.md        # 调度快照
├─ memory-snapshot.md      # 最近章节摘要
├─ qidian-analysis.md      # 起点热门小说分析报告
├─ 人物/<角色名>.md        # 角色卡
├─ 伏笔/foreshadow-xxx.md  # 伏笔追踪
├─ 章节/chapter-xxx.md     # 章节工作文件
└─ <书名 slug>/第X卷/      # 纯正文目录
```

## 交付汇报模板

每次完成操作后汇报：
1. 当前状态和进度
2. 本轮做了什么（写了哪章、改了什么）
3. 字数统计（本章字数 / 累计字数 / 目标字数）
4. 使用的写作技法
5. 记忆写入情况（伏笔、角色变更、摘要）
6. 下一步操作
