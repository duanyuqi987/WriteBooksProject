# 章节同步与记忆写入

review 通过后的收尾工作。把章节产出中的变化写入 6 个目标文件，构建持续记忆。

## HARD-GATE

- 除非 novel-review 明确判定当前章为"通过"（pass），否则不得调用本 skill
- 不得做任何创作判断——本 skill 仅将事实同步到系统文件
- 6 项同步缺一不可

## 6 项同步检查清单

### 1. 读取上下文

读取以下文件（只读）：
- `章节/chapter-xxx.md`（正文 + 技法自检 + frontmatter）
- `project.md`（当前 canon）
- `outline.md`（当前状态）
- `人物/` 所有角色卡
- `伏笔/` 所有伏笔文件
- `memory-snapshot.md`（当前记忆快照）
- `project-state.md`（当前调度状态）

### 2. 更新 project.md canon

检查本章是否有设定变更：
- 新角色？→ 提示创建角色卡
- 新地点/组织/物品？→ 追加到 project.md 变更日志
- 能力/状态变化？→ 追加到 project.md 变更日志
- 世界规则揭示/修正？→ 追加到 project.md 变更日志

**格式**：`[chapter-XXX] 变更类型：具体内容`

### 3. 更新角色卡

检查本章中每个出场角色的状态变化：
- 能力变化？→ 追加到角色卡「变更记录」
- 关系变化？→ 追加到角色卡「变更记录」
- 心态/驱动力/恐惧变化？→ 追加到角色卡「变更记录」
- 关键事件影响？→ 追加到角色卡「变更记录」

**格式**：`[chapter-XXX] 变化内容描述`

### 4. 更新伏笔状态

根据章节 frontmatter 中的 `foreshadows_planted` 和 `foreshadows_paid`：

**新埋入的伏笔（foreshadows_planted）：**
- 如果不存在对应 `伏笔/foreshadow-xxx.md` → 创建新文件
- 更新状态为 `planted`
- 记录埋入章和埋入描述

**已兑现的伏笔（foreshadows_paid）：**
- 更新对应伏笔文件状态为 `paid_off`
- 记录兑现章和兑现描述

**中途提示的伏笔（如果有）：**
- 追加到对应伏笔文件的「中途提示」列表
- 更新状态为 `reminded`

### 5. 更新章节文件与大纲状态

- 在章节文件中写入**定稿摘要**（2-3 句话）：关键事件 + 角色变化 + 新元素
- 将本章 frontmatter status 改为 `done`
- 更新 `outline.md` 中本章状态为 `done`
- 检查当前卷是否全部 done → 如果是，标记卷完成

### 6. 更新记忆快照与调度状态

**更新 memory-snapshot.md：**
- 追加本章摘要（格式见 `skills/novel-writing/references/memory-system.md`）
- 保持最近 5 章详细摘要，更早的合并为 1 句话
- 更新「当前进度」section（当前章、总字数、未兑现伏笔数、活跃角色列表）
- 更新「最近更新时间」

**更新 project-state.md：**
- 状态更新为下一章的状态（planned 或本卷 done）
- 当前章更新为下一章
- 记录上次操作描述
- 更新上次会话时间

## 字数统计与进度汇报

每次 update 后汇报：

```markdown
## 本章完成汇报

- **章节：** chapter-XXX
- **字数：** XXXX 字（目标 XXXX 字，完成率 XX%）
- **累计字数：** XXXX 字 / 目标 XXXX 字（总完成率 XX%）
- **剩余章节：** XX 章 / 共 XX 章
- **本章技法：** XXX
- **本章伏笔：** 埋入 N 条 / 提示 N 条 / 兑现 N 条
- **未兑现伏笔总数：** N 条
- **角色状态变化：** [有/无]（如有，列出）
- **Canon 变更：** [有/无]（如有，列出）
- **下一章：** chapter-XXX - [任务说明]
```

## 核心原则

- **只追加不修改**：初始设定不修改，只追加变更日志
- **事实性同步**：只记录事实，不做评价
- **6 项同步缺一不可**：即使某项"无变化"也要确认
- **伏笔文件即真相**：伏笔状态以伏笔文件为准
- **记忆快照保持紧凑**：只保留最近 5 章详情

## 反模式

| 错误 | 正确 |
|------|------|
| 修改 project.md 初始设定 | 只追加变更日志 |
| 跳过伏笔文件创建 | 每条伏笔必须有独立文件 |
| 忘记更新 memory-snapshot | 每次 update 必须更新快照 |
| 角色变化不记录 | 每次角色变化追加到角色卡 |
| 不检查卷是否完成 | 每次检查，卷完成时明确标记 |
| 在变更日志中写评价 | 只记录事实 |
