# HTML 学习书数学公式写法规范

本文件规定所有 HTML 学习书中数学公式的写法规范。以 OMPL 学习书的公式风格为参照标准，要求每个公式块**自包含**——读者在公式块内部就能完整理解公式含义，不需要跳到外部段落寻找解释。

## 零、MathJax 3 配置（必须）

每个 HTML 学习书的 `<head>` 中必须包含以下 MathJax 3 配置：

```html
<script>
  window.MathJax = {
    tex: {
      inlineMath: [['\\(', '\\)']],
      displayMath: [['\\[', '\\]']],
      processEscapes: true
    },
    options: {
      skipHtmlTags: ['script', 'noscript', 'style', 'textarea', 'pre', 'code']
    }
  };
</script>
<script defer src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-chtml.js"></script>
```

关键约束：
- 行内公式统一用 `\(...\)`，禁止使用 `$...$`
- 显示公式统一用 `\[...\]`，禁止使用 `$$...$$`
- `processEscapes: true` 必须开启，确保 `\\(` 和 `\\)` 正确解析
- CDN 固定使用 `tex-chtml.js`，不切换到 SVG 输出

## 一、CSS 规范（必须）

### 1.1 公式块基础样式

公式块与示例框、提示框共享基础盒模型样式，再叠加等宽字体：

```css
.example-box, .num-example, .callout, .formula-block, .chapter-note {
  margin: 18px 0;
  padding: 12px 14px;
  border: 1px solid var(--border);
  border-left: 4px solid var(--accent);
  border-radius: 6px;
  background: color-mix(in srgb, var(--panel) 88%, var(--bg));
}

.formula-block {
  font-family: ui-monospace, SFMono-Regular, Menlo, Consolas, monospace;
  white-space: pre-wrap;
}
```

关键约束：
- `.formula-block` 必须以 `font-family: ui-monospace, ...` 设置等宽字体
- `.formula-block` 必须设置 `white-space: pre-wrap`，保留内部换行
- 左边框强调色（`border-left: 4px solid var(--accent)`）是公式块的视觉标识
- 不要为公式块单独定义完全不同的盒模型；它应与 `.num-example`、`.callout` 共享基础样式

### 1.2 MathJax 渲染溢出处理

```css
mjx-container[jax="CHTML"][display="true"] {
  overflow-x: auto;
  overflow-y: hidden;
  padding: 4px 0;
}
```

作用：当公式宽度超出容器时，出现水平滚动条而非溢出破坏布局。

### 1.3 暗色主题

```css
[data-theme="dark"] {
  --text: #e8edf2;
  /* ... 其他暗色变量 */
}
```

MathJax 渲染的公式会自动继承 `--text` 颜色。不需要为 `mjx-container` 单独设置暗色颜色——它通过 CSS 变量继承。如果暗色主题下公式颜色异常，检查 `--text` 变量是否正确。

## 二、核心规则：公式块必须自包含

**Golden Rule：公式块的 `<div>` 内部必须包含理解该公式所需的全部描述——公式是什么、每个符号的含义、直观解释。读者不应需要跳到外部 `<p>` 标签才能理解公式。**

### 好例子（自包含公式块）

```html
<div class="formula-block">
<strong>回报（Return）G_t —— 从时刻 t 开始，智能体在未来累积获得的全部奖励：</strong>

回报不是"即时奖励"，而是"从当前到未来的奖励总和"，每往未来走一步，奖励的重要性就打一个折扣 γ（0 ≤ γ ≤ 1）。

\[
G_t = R_{t+1} + \gamma \cdot R_{t+2} + \gamma^2 \cdot R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k \cdot R_{t+k+1}
\]

直观理解：如果把每个时刻的奖励看作"收入"，G_t 就是"从今天开始所有未来收入的现值总和"——未来的收入要乘以贴现因子 γ^k。
</div>
```

这个块包含三部分：
1. **标题行**（`<strong>`）：公式名称 + 一句话定义
2. **公式**（`\[...\]`）：LaTeX 显示公式
3. **直观解释**：用日常语言解释公式的直觉含义

### 反例（禁止）

```html
<!-- 反例1：裸公式，没有描述 -->
<div class="formula-block">
\[
G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}
\]
</div>

<!-- 反例2：描述在外部 p 标签中，公式块内只有公式 -->
<p>回报 G_t 是从时刻 t 开始的累积折扣奖励。</p>
<div class="formula-block">
\[
G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}
\]
</div>

<!-- 反例3：使用单独的 label span，描述和公式分离 -->
<div class="formula-block">
<span class="formula-label">回报 G_t</span>
\[
G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}
\]
</div>
```

## 三、公式块的六种模式

### 模式 1：标题 + 描述 + 公式 + 解释（最完整）

适用场景：核心定义、重要定理、需要深入理解的公式。

```html
<div class="formula-block">
<strong>公式名称 —— 一句话定义：</strong>

用 1-2 句自然语言描述公式的直觉含义。说明关键符号的含义和取值范围。

\[
公式的 LaTeX 表示
\]

直观理解/补充说明：用日常类比或场景帮助读者建立直觉。
</div>
```

### 模式 2：定义文本 + 公式（适合数学定义）

适用场景：数学对象的定义，文字和公式紧密交织。

```html
<div class="formula-block">
定义：集合名 = { 元素 | 约束条件 }

每个元素的参数化表示：
\[
R(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix}
\]

其中参数满足的条件说明。
</div>
```

注意：文字定义部分放在 `\[...\]` 之前，作为公式的前导说明。

### 模式 3：多个公式共用一个块

适用场景：一组紧密关联的公式，同属一个概念。

```html
<div class="formula-block">
四元数的定义：
\[
q = w + xi + yj + zk = (w, x, y, z)
\]
满足约束：w² + x² + y² + z² = 1（单位四元数）

其中 i² = j² = k² = ijk = -1

绕单位轴旋转的表示：
\[
q = \left(\cos\frac{\theta}{2}, u_x\sin\frac{\theta}{2}, u_y\sin\frac{\theta}{2}, u_z\sin\frac{\theta}{2}\right)
\]
</div>
```

规则：
- 同一个概念下的多个公式放在同一个 `.formula-block` 中
- 公式之间用文字说明连接
- 不要为每个公式创建独立的块——那样会切断它们之间的逻辑关联

### 模式 4：纯文本公式块（无需 LaTeX）

适用场景：问题形式化定义、算法步骤、结构化描述。

```html
<div class="formula-block">
问题的形式化定义：

给定：
  - 输入空间 X
  - 输出空间 Y
  - 训练样本集 D = {(x₁, y₁), ..., (xₙ, yₙ)}

找到：
  - 一个函数 f: X → Y
  - 使得在未知样本上的泛化误差最小

如果不存在满足误差要求的函数，则报告"不可学习"。
</div>
```

规则：
- 纯文本公式块也使用 `.formula-block` 类
- 用结构化缩进（`给定：`/`找到：`）表达数学定义
- 这是公式块的合法形式，不强制要求包含 LaTeX

### 模式 5：数值示例（`.num-example`）

适用场景：用具体数字演示公式的计算过程。

```html
<div class="num-example">
<strong>数值例子：</strong>场景描述。设参数 A = 3, B = 4, C = 5，代入公式：
\[
\text{结果} = \sqrt{3^2 + 4^2} = \sqrt{9 + 16} = \sqrt{25} = 5
\]
计算结果表明：（用自然语言解释结果的含义和直觉）。
</div>
```

规则：
- 数值示例使用 `.num-example` 类，不是 `.formula-block`
- 开头必须标注 `<strong>数值例子：</strong>` 或 `<strong>例子：</strong>`
- 必须包含具体数值和完整的计算过程
- 结尾必须用自然语言解释计算结果的含义
- `.num-example` 不需要等宽字体，使用正文字体即可

### 模式 6：行内公式

适用场景：在正文中嵌入简短数学符号或表达式。

```html
<p>两个状态之间的距离通常用<strong>欧氏距离（L2 范数）</strong>来度量：</p>
```

规则：
- 在正文段落中使用 `\(...\)` 嵌入行内公式
- 行内公式只用于简短的符号引用（如 `\(\mathcal{C}\)`、`\(\mathbb{R}^n\)`）
- 完整的重要公式应放在 `.formula-block` 中使用 `\[...\]`
- 不要在正文段落中用行内公式展示复杂推导

## 四、写法指南

### 4.1 公式块内文字的语气

- 使用自然中文叙述，像在给读者当面讲解
- 用"如果把……看作……"、"想象一下……" 这类引导语建立直觉
- 解释"为什么需要这个公式"，而不只是复述"公式是什么"
- 用"直观理解："、"换句话说："开头写解释段落

### 4.2 符号说明

- 公式中每个引入的新符号必须在同一公式块内说明
- 符号说明格式：`符号（含义，取值范围，约束条件）`
- 例：`γ（折扣因子，0 ≤ γ ≤ 1，控制未来奖励的重要性）`

### 4.3 推导过程

- 关键推导步骤可以在一个公式块中使用多个 `\[...\]` 展示
- 每个推导步骤之间用文字说明"这一步做了什么"
- 12 步完整推导可以全放在一个 `.formula-block` 中

### 4.4 公式块的数量控制

- 不要为了"覆盖率"给每个简短公式都补一大段解释
- 附录中的参考公式、论文引用块、简单符号定义等可以保持简洁
- 判断标准：公式块内的文字是否增加了读者对公式的理解？如果没有，保持简洁即可

## 五、反模式清单

以下写法**禁止**出现在最终 HTML 中：

| 反模式 | 说明 | 正确做法 |
|--------|------|----------|
| 裸公式 | `.formula-block` 中只有 `\[...\]`，没有任何文字 | 至少加一行公式名称和一句话解释 |
| 标签分离 | 用 `<span class="formula-label">` 单独标注公式名 | 用 `<strong>` 直接在块内写标题 |
| 外部描述 | 公式的解释写在外部 `<p>` 标签中 | 将解释移入 `.formula-block` 内部 |
| 孤立符号 | 公式中出现未说明的符号 | 在公式前后的文字中说明每个符号 |
| 无解释数值 | `.num-example` 中只有计算，没有结果解释 | 在计算结果后加一句自然语言解释 |
| 公式堆砌 | 一个块中出现 5+ 个不相关的公式 | 按概念拆分成多个块 |
| 正文长推导 | 在 `<p>` 标签中用行内公式写多步推导 | 推导放入 `.formula-block` 中使用 `\[...\]` |
| 缺少暗色适配 | 暗色主题下公式颜色不跟随 `--text` 变量 | 使用 CSS 变量，不写死颜色值 |

## 六、与其他规则的关联

- 本文件是 `必读写书规则.md` 的补充，专注于数学公式的特殊写法要求
- 公式块的 CSS 变量（`--bg`、`--panel`、`--text`、`--border`、`--accent`）定义在 HTML 的 `:root` 和 `[data-theme="dark"]` 中，必须与整体配色一致
- 公式块内的中文写作风格遵循 `references/写作方法/` 中的长文可读性要求
- 字数统计脚本 `scripts/count_text.py` 会统计公式块内的中文和英文文本，不算公式的 LaTeX 源码

## 七、检查清单

在宣称书稿完成前，逐项确认：

- [ ] MathJax 3 配置正确（`\(`/`\[` 分隔符，`processEscapes: true`，CDN 指向 `tex-chtml.js`）
- [ ] `.formula-block` CSS 包含等宽字体和 `white-space: pre-wrap`
- [ ] `mjx-container[jax="CHTML"][display="true"]` 设置了 `overflow-x: auto`
- [ ] 暗色主题下公式颜色正常
- [ ] 核心公式块全部自包含（描述 + 公式 + 解释在同一 div 内）
- [ ] 没有裸公式（公式块内至少有一行文字说明）
- [ ] 没有使用 `$...$` 或 `$$...$$` 作为数学分隔符
- [ ] 数值例子使用 `.num-example` 类，开头标注"数值例子："
- [ ] 行内公式只在简短引用时使用，复杂推导已移入公式块
