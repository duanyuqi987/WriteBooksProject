# 写书前必读规则

这份文件是 `project-to-learning-doc` 每次写书、续写、重写、更新书稿前必须先读的规则。它的作用是把项目固定成“中文 HTML 学习书生产线”，防止输出跑成教案、提纲、Markdown 或资料摘抄。

## 1. 最终产物只能是 HTML

- 最终书稿必须是单个 `.html` 文件。
- 不把最终书稿写成 Markdown、PDF、DOCX、纯文本、教案表格或多文件页面。
- 允许在项目内部使用 Markdown 维护规则、说明和迭代记录；但交付给用户学习的书稿只能是 HTML。
- 默认模板是 `assets/templates/learning-document.html`。

## 2. 书名必须是中文

- HTML `<title>` 必须是中文书名。
- 正文首个 `<h1>` 必须是同一个中文书名或包含这个中文书名。
- 来源是英文书、英文网站、英文代码库时，也要把书名翻成自然中文。
- 文件名可以使用英文 slug，便于系统路径稳定；文件名不等于书名。

推荐：

```text
书名：OMPL 运动规划算法学习书
文件：docs/2026-05-13/学习书/ompl-motion-planning-learning-book-2026-05-13.html
```

## 3. 用户要求多少字，就必须达到多少字

- 用户指定 `N 字`、`N 万字`、`不少于 N 字` 时，目标字数就是硬性下限。
- 每轮写作后运行 `python3 scripts/count_text.py <html文件>`，以 `estimated_total` 作为本项目的本地验收口径。
- `estimated_total < 用户目标字数` 时，只能汇报“当前进度”，不能宣称“已经完成整本书”。
- 单轮写不完时，继续按章追加，直到达到或超过目标字数。
- 不得用源码、网页原文、目录、样式、脚本或重复模板凑字数。

## 4. 日期目录与中文功能目录

后续新书稿和新迭代产物默认按日期进入 `docs/` 下的中文功能目录：

```text
docs/
  YYYY-MM-DD/
    学习书/      # 最终交付给用户阅读的 HTML 书稿
    迭代记录/    # 当天修改记录、验证记录、阶段记录
    来源缓存/    # 网站、资料、截图、页面缓存等来源副本
    构建脚本/    # 与当天书稿强相关的一次性构建脚本
    质量检查/    # 字数统计、结构检查、截图检查等结果
```

如果用户正在打开旧路径，不要强行移动旧文件；可以从下一次新产物开始使用日期目录，并在汇报中说明新旧路径。

## 5. 写作前必须先建立来源地图

所有“根据资料写书”的任务都必须先读资料再写正文。来源地图至少说明：

- 来源类型：书、网站、代码库、项目文件夹、论文、日志、混合资料。
- 来源语言：中文、英文、双语、代码/配置。
- 来源可信度：官方、一手资料、项目内部资料、二手资料、未知。
- 来源版本或访问日期。
- 可引用范围：可短引、可转述、可链接、只可结构参考。
- 缺口和冲突。

未读取的资料不能伪装成已读取；无法访问的资料要明确标为缺源。

## 6. 书稿正文必须像书

当用户说“写成书”“像书一样”“我是小白”“让我学会”“讲清楚算法和代码”时，正文必须使用书籍式连续讲解：

- 用章节叙事，而不是任务卡片。
- 先讲直觉，再讲正式定义。
- 抽象概念后必须给例子、图、公式、伪代码或代码对应。
- 代码必须解释输入、输出、关键变量、设计原因和改错影响。
- 算法必须讲问题定义、步骤、手工推演、适用场景、失败模式和参数。
- 每章要有自然过渡，不要机械堆“学习目标、课堂活动、教师提示、学生练习”。

需要更细的写法时，按任务读取 `references/写作方法/`：

- `范文参照.md`：单个知识点怎样写完整。
- `伏笔管理.md`：怎样管理概念铺垫和前后呼应。
- `网文技法.md`：怎样提升长文可读性。
- `修订体系.md`：怎样维护版本、修订日志和稳定 ID。
- `经验总结.md`：怎样避开历史失败模式。

## 7. 中英文资料统一写成中文书

- 英文术语第一次出现时使用“中文译名（English Term, 缩写）”。
- 同一术语全书保持同一译法。
- 保留必要英文原名、API 名、类名、函数名、配置键和路径。
- 不整页翻译英文网站，不复刻书籍原文；要重构成中文学习路径。

## 8. 更新旧 HTML 时保持稳定

- 先读取旧 HTML，找到目标章节稳定 ID。
- 只替换目标章节，不无关重排。
- 更新目录、`last-update`、`word-count` 和修订日志。
- 更新后再次运行字数统计和结构检查。

## 9. 完成前检查

宣称完成前至少确认：

- 目标文件是 `.html`。
- 书名是中文，且出现在 `<title>` 和首个 `<h1>`。
- 用户指定目标字数时，`estimated_total` 已达到或超过目标。
- 来源地图覆盖主要资料。
- HTML 有目录、正文、版本信息、来源说明、修订日志和稳定章节 ID。
- 没有把学习书写成教案模板、资料卡片或原文搬运。
