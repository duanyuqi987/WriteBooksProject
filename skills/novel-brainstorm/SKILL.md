# 小说头脑风暴

把散乱灵感和目标字数收敛成能开工的 `project.md`、`人物/` 文件夹和读者体验规格。

## HARD-GATE

- 在 project.md 写好且用户确认之前，不得调用其他 novel skill
- 每一轮对话必须只包含一个问题，附带 2-4 个选项
- 首轮必须先确认字数规模

## 15 维度渐进收束

### 第 0 步：字数规模确认（首轮必做）

**问题**："你打算写多少字的小说？"

**选项**：
1. 10 万字（短篇，40-50 章，1-2 卷）
2. 30 万字（中篇，120-150 章，2-3 卷）
3. 50 万字（中长篇，200-250 章，3-5 卷）
4. 80 万字及以上（长篇，320+ 章，4-6 卷）

### 14 维度渐进收束

1. **字数规模** — 锁定全书字数和每章目标（首轮确认）
2. **灵感采集** — 记录用户说的所有内容，不做评判
3. **题材与气质** — 锁定故事类型和氛围
4. **第一性原理挑战** — 挑战题材惯例，找到核心价值（新增）
5. **读者体验规格** — 定义 2-3 个核心感受和人的时刻（新增）
6. **主题** — 锁定故事最终想探讨什么
7. **主角与关系网** — 锁定主角、配角、叙事视角、名字
8. **角色工坊** — 用身份张力公式设计主角和关键配角（新增）
9. **世界观** — 锁定世界类型和核心规则（3-7 条）
10. **核心冲突** — 锁定主线冲突和长篇驱动机制
11. **竞争性情节** — 生成 2 种替代情节方向，用户选择（新增）
12. **开篇策略** — 锁定故事如何开始
13. **写作风格** — 锁定文字风格 + 叙事者声纹
14. **技法偏好** — 从 10 种技法中选择 2-3 个核心技法
15. **非目标** — 锁定不写什么

## 新增维度详解

### 第 4 步：第一性原理挑战

在锁定题材后，挑战题材惯例：

**问题**：以"这本小说的核心价值是什么？"为起点，挑战所有"这个题材应该这样写"的假设。

**挑战模板**：
```
这个题材的惯例 → 为什么存在 → 我们能不做吗 → 替代方案
```

**示例**（修仙题材）：
- 惯例：主角必须从废柴开始 → 为什么要废柴？ → 如果主角一开始就是天才但背负着不能修炼的秘密呢？
- 惯例：必须有等级体系（炼气→筑基→金丹→元婴）→ 为什么要等级？ → 如果力量增长是代价驱动的（每次突破都需要牺牲一样东西）？

**目的**：确保故事不是因为"大家都这么写"而写成这样。

### 第 5 步：读者体验规格

**问题**："你希望读者在读这部小说时，最核心的 2-3 个感受是什么？"

**选项**（选 2-3 个）：
1. **放不下** — 每章结束都想继续读（章末钩子要强）
2. **真实** — 角色像真人，世界可信（白描功力要强）
3. **聪明** — 情节设计尊重读者智商（伏笔要有层次）
4. **心疼** — 对角色的情感依附（角色弧光要有深度）
5. **震撼** — 某些场景的冲击力久久不散（高潮要重投入）
6. **意外** — 没想到但回想起来很合理（反转要有铺垫）
7. **沉浸** — 读完才发现过了一小时（细节密度要高）
8. **温暖** — 残酷世界中的一点点亮色（不能全程黑暗）

然后定义至少 1 个"人的时刻"：
```
时刻：[描述性名称]
场景：什么时候/在哪里/什么状态
感受：读者应该感受到什么
```

### 第 8 步：角色工坊

使用身份张力公式（详见 `skills/novel-writing/references/character-creation.md`）：

```
身份张力 = 前世身份 × 当下处境 × 内在矛盾
```

**问题**："关于主角，请回答三个问题：ta 从哪里来？ta 现在处于什么困境？ta 最核心的内在矛盾是什么？"

每个问题附带 2-4 个选项。完成后生成一句话灵魂。

### 第 11 步：竞争性情节

深度融入 iPhone 团队"内部竞争"方法论。

在核心冲突确定后，生成 2 种替代的情节方向：

**选项 A（演化型）**：在题材惯例上的精炼优化
**选项 B（革命型）**：完全不同的处理方式

让用户盲选，不告知哪个是演化哪个是革命。只描述阅读体验的差异。

**示例**（修仙题材）：
- 方向 A："一个废柴通过层层闯关成长为强者"——经典爽文路径，读者熟悉节奏
- 方向 B："一个强者伪装成废柴潜入宗门，却在伪装中发现了比力量更重要的东西"——颠覆预期

## project.md 新增字段

- **体验规格**：核心感受 + 人的时刻
- **第一性原理分析**：挑战了哪些惯例
- **竞争性情节选择**：选择的方案 + 落选方案的精华

## 交付

1. 展示完整 project.md（含新增字段）
2. 展示体验规格
3. 创建人物/ 文件夹（含声纹档案的角色卡）
4. 用户确认后写入 project-state.md，路由到 novel-outline
