# 正文起草

本工序负责写正文、续写、改写和局部润色。写作前先判断任务是完整章节、短篇全文、场景片段还是文字改造，然后选择合适流程。

## 启动读取

标准项目和长篇连续项目必须读取：

1. `outline.md`：当前章节或场景任务。
2. `project.md`：题材、主题、canon、风格、禁用项。
3. `memory-snapshot.md`：最近章节摘要。
4. `人物/`：本章出场角色卡。
5. `伏笔/`：本章相关伏笔。
6. 前一章末尾或上文片段：保证衔接。
7. `skills/novel-writing/references/ai-flavor-banlist.md`：去 AI 味检查。
8. `skills/novel-writing/references/chapter-rhythm.md`：章节和场景节奏。
9. 相关风格文件：如 `通用中文小说.md`、`冷白描.md`、`怪诞悬疑.md`、`系统爽文.md`。
10. 如果用户要求自动批量写章、完整短篇包或 InkOS 流水线，读取 `skills/novel-writing/references/inkos-integration.md` 并路由到 `skills/inkos-bridge`。

轻量片段任务可以只读取用户提供的上下文，但必须说明缺失信息带来的假设。

## 写作流程

### 1. 场景定位

写之前先用 3 句话说明内部目标：

- 这个场景/章节要改变什么？
- 角色在这里想要什么，阻力是什么？
- 读者读完后应多知道什么或多感到什么？

### 2. 初稿

按场景顺序写清楚：

- 谁在场
- 发生了什么
- 角色做了什么选择
- 选择带来什么后果
- 结尾留下的是余味、疑问、转折还是安静落点

长篇和连载章节可吸收 InkOS 的两阶段纪律：先写正文变化，再做状态结算。正文里必须看得见人物动作、物件、对话或事件，不允许只在备注里宣布“关系变化”“伏笔推进”。

### 3. 二次打磨

重点处理：

- 对话是否像这个角色会说的话
- 叙述是否符合项目风格
- 信息是否过早解释或重复解释
- 抽象形容是否能改成动作、细节、场面
- 节奏是否有快慢变化

### 4. 技法微调

只选择本章真正需要的 1-3 个技法。技法服务故事，不为了展示技巧而堆叠。

常用选择：

- 白描：人物行动、现实细节、场面落地。
- 不写之写：关键情绪、创伤、秘密、余味。
- 草蛇灰线：伏笔和线索。
- 一击两鸣：一个场景同时推进情节和关系。
- 暗透法：让旁人说出角色不愿说破的事。
- 繁章波折法：冲突场景内多次转折。

## InkOS 伏笔账本纪律

如果本章沿用 InkOS 的 hook ledger 或本项目 `伏笔/` 文件：

- `mention`：正文自然触碰伏笔，不推进结果。
- `advance`：伏笔必须在正文中有可定位的推进场景。
- `resolve`：伏笔兑现必须有具体动作、物件、对话或事件支撑。
- `defer`：只在章末备注记录延后理由，不在正文里假兑现。

任何 `advance` / `resolve` 都不能只写在章节备注或记忆同步里。

## 章末处理

按作品取向选择，不强制所有小说都用悬念钩子：

| 取向 | 章末可用处理 |
|---|---|
| 连载/类型 | 行动钩、发现钩、选择钩、危机钩 |
| 文学/现实 | 动作落点、意象回响、情绪余波 |
| 悬疑/推理 | 新线索、旧线索变义、误导加深 |
| 言情/关系 | 关系错位、未说出口的话、姿态变化 |
| 短篇 | 余味、反照、轻微开放结局 |

禁止用空泛总结、作者替读者解释主题、机械预告下一章。

## 产出

完整章节默认产出两个文件：

- `章节/chapter-xxx.md`：章节工作文件，含目标、正文、字数、待审项。
- `正文/chapter-xxx.md` 或项目约定的纯正文路径：只保留正文。

章节 frontmatter 建议：

```yaml
---
id: chapter-001
volume: 1
status: reviewing
review_round: 0
word_count_target: 2500
word_count_actual: 0
techniques_used: []
foreshadows_planted: []
foreshadows_paid: []
canon_changed: false
---
```

## 自检

交付前至少检查：

- 本章是否完成大纲任务。
- 是否出现 canon 冲突。
- 角色对话是否可区分。
- 是否有空泛、套路化、AI 腔句子。
- 章节结尾是否符合本作品取向。
- 字数是否明显偏离目标；偏离时说明原因。
- 伏笔推进或兑现是否在正文里有实际落点。
