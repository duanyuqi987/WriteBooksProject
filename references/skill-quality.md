# Skill 质量优化参考

## 来源与用途

本参考用于优化 `project-to-learning-doc` 这类流程型 skill。参考来源包括 `skills.sh` 的公开目录、优秀 skill 页面和本地 `writing-skills` 规则。目标不是照抄其他 skill，而是吸收它们的结构原则，让本 skill 更容易被 AGENT 正确触发、正确执行、正确验证。

## 优秀 skill 的共性

1. **description 只写触发条件**
   - 用 `Use when...` 开头。
   - 描述用户会说什么、任务有什么症状、何时应该加载该 skill。
   - 不把完整工作流程塞进 description，避免 AGENT 只看摘要就跳过正文。

2. **正文先给核心判断**
   - 开头要让 AGENT 迅速判断当前任务属于哪一类。
   - 对容易混淆的任务写清“什么时候用”和“什么时候不用”。
   - 对本 skill 来说，最重要的分歧是：这本 HTML 学习书应采用算法学习、源码导读、网站知识整理、书籍学习指南、教师用书还是混合素材书作为主线。

3. **复杂内容拆到 references**
   - `SKILL.md` 保持短、清晰、可执行。
   - 长规则、类型蓝图、HTML 模板、质量检查放到 `references/` 或 `assets/`。
   - AGENT 只在需要时读取相关 reference，避免每次加载过多内容。

4. **把失败模式写成红线**
   - 比如“写成模板卡片”“没读来源就长篇发挥”“把网页翻译成替代原文”“代码讲解变源码粘贴”。
   - 红线要可检查，最好能配合 `rg`、字数统计或结构检查脚本。

5. **能自动验证的就自动验证**
   - 字数估算用脚本。
   - HTML 生成后检查标题、目录、来源、修订日志。
   - 用户明确禁止的表达要用搜索命令检查。
   - skill 优化后运行 quick validate，确认 frontmatter 和文件结构有效。

## 本 skill 的优化重点

### 触发词覆盖

description 和正文要覆盖这些真实用户说法：

- “写成书”
- “像书一样”
- “我是小白”
- “10 万字 HTML”
- “根据网站/书/代码库/项目文件夹写学习文档”
- “一个个理解算法和代码”
- “不要教案”
- “不要学习目标模板”
- “优化这个 skill”

### 书内主线选择

| 用户说法 | 应选择的模式 | 不应默认加入 |
|---|---|---|
| 写成书、像书一样、小白理解算法和代码 | 学习书 | 教师提示、课堂流程、学生练习 |
| 给我上课、写教案、按课时安排 | HTML 教师用学习书 | 纯散文式长章 |
| 看懂项目、看懂代码库 | HTML 源码学习书 | 泛泛概念文 |
| 根据英文网站整理 | HTML 网站知识学习书 | 逐页翻译 |
| 根据书学习 | HTML 书籍学习指南 | 可替代原书的复刻 |

无论采用哪条主线，最终交付都必须是中文书名的单文件 HTML。

### 学习书质量红线

学习书模式下应满足：

- 标题像书，不像任务清单。
- 正文是连续讲解，不是固定卡片。
- 对算法必须有直觉、公式或伪代码、图、数值例子、代码对应。
- 对代码必须解释输入、输出、设计原因、关键变量和改错影响。
- 对网站必须保留来源链接和访问日期。
- 对书籍必须原创讲解，不能复刻原文。

学习书模式下避免：

- 把每节写成“学习目标、前置知识、常见坑、复习问题”。
- 默认写“教师提示、课堂活动、学生练习”。
- 为凑字数粘贴大段源码、网页或书籍原文。
- 只按原资料目录排列，不重构学习路径。

## 从 skills.sh 借鉴的检查流程

1. 打开 `https://www.skills.sh/`，按主题或排行榜找同类优秀 skill。
2. 优先看安装量、仓库来源、是否有安全审计和 SKILL.md 摘要。
3. 只抽象结构原则，不复制对方正文。
4. 回到本项目，对照检查：
   - description 是否是触发条件。
   - `SKILL.md` 是否短而可执行。
   - references 是否分层清楚。
   - 是否有脚本或命令验证输出质量。
   - 是否写清不该触发和不该输出的情况。
5. 修改后运行验证命令，并把结果记入当天迭代文档。

## 建议验证命令

```bash
python3 /home/ps/.codex/skills/.system/skill-creator/scripts/quick_validate.py /home/ps/public/project-to-learning-doc
rg -n "教师提示|课堂活动|学生练习|学习目标：|复习问题：" SKILL.md references 使用说明.md
python3 scripts/count_text.py docs/ompl-100k-learning-2026-05-13.html
```

命令命中不一定都代表错误；如果是在“不要这样写”的红线说明里，可以保留。但生成给用户的学习书正文不应命中这些旧式模板表达。
