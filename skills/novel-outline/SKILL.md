# 小说大纲规划

把 `project.md` 转化为可执行的叙事路线。大纲粒度必须适配作品形态：短篇不要硬拆几十章，长篇不要只写几句方向。

## 硬规则

- 未读取 `project.md` 不开始规划。
- 先确定作品形态，再选择结构模型。
- 大纲服务小说，不为了表格完整而牺牲故事自然性。
- 长篇只精细规划当前推进区间，后续保留方向种子。

## 结构模型

| 作品形态 | 推荐结构 | 产出 |
|---|---|---|
| 短篇 | 起势 → 转折 → 逼近 → 落点 | 关键场景表，不强制章节 |
| 中篇 | 三幕 / 四幕 / 章节组 | 幕目标 + 章节路线 |
| 长篇 | 部 / 卷 / 章节 | 卷结构 + 当前卷详细章节 |
| 连载 | 长线目标 + 当前卷 + 滚动 5-10 章 | 追踪状态、伏笔、节奏窗口 |

## 规划步骤

1. **读取项目底座**：premise、主题、角色、世界规则、读者体验、禁用项。
2. **选择结构模型**：短篇、中篇、长篇或连载。
3. **确定叙事主线**：主角欲望、阻力、代价、不可逆变化。
4. **安排人物弧线**：主角和关键配角在每个阶段的状态变化。
5. **分配信息披露**：秘密、伏笔、误导、反转、真相的出现顺序。
6. **生成章节或场景路线**：每一项必须有叙事功能。
7. **标注节奏与技法**：按场景需要选择，不强行堆技法。
8. **留下待定项**：不确定的设定不要伪装成已确认 canon。

## 章节/场景功能

每个章节或关键场景至少承担一项功能：

- 推进主线冲突
- 改变人物关系
- 揭示新信息
- 埋入、提示或兑现伏笔
- 让角色做出选择并付出代价
- 改变读者对人物或世界的理解

删掉后没有损失的章节，不进入大纲。

```markdown
|---|---|---|---|---|

| 转折 | 打破平衡 |  |  |  |
| 逼近 | 压力升级 |  |  |  |
| 落点 | 完成余味 |  |  |  |
```

### 中长篇

```markdown
## 全书结构
| 阶段/卷 | 篇幅 | 本阶段任务 | 起点状态 → 终点状态 |
|---|---:|---|---|

## 当前推进区间
| 章节 | 任务说明 | 结构标记 | 关键变化 | 伏笔操作 | 状态 |
|---|---|---|---|---|---|
| chapter-001 |  | setup |  | plant/remind/pay/- | planned |

## 精细规划
### chapter-001：[暂定标题]
- 目标：
- 场景拆分：
- 人物变化：
- 信息披露：
- 风格与技法：
- 章末处理：
```

## 滚动展开

长篇和连载使用滚动窗口：

- 当前 5-10 章：精细规划。
- 下一阶段：一句话章节路线。
- 更远部分：只保留方向种子和必须兑现的大伏笔。

写作偏离原计划时，允许更新后续大纲，但必须记录原因。

## 交付

1. 展示采用的结构模型和理由。
2. 给出大纲正文。
3. 标出待确认问题和风险点。
4. 用户确认后保存 `outline.md`，并更新 `project-state.md`。
