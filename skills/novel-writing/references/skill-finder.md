# 写作技法查找

根据用户请求和当前创作阶段，快速匹配写作技法、风格文件和下一步工序。

## 查找流程

```text
用户请求
  ↓
识别任务：构思 / 大纲 / 正文 / 审查 / 修改 / 同步
  ↓
识别作品取向：现实 / 文学 / 悬疑 / 言情 / 历史 / 幻想 / 网文 / 未指定
  ↓
识别章节功能：开篇 / 冲突 / 过渡 / 高潮 / 余波 / 伏笔
  ↓
返回：工序 + 风格文件 + 1-3 个技法
```

## 按用户意图路由

| 用户请求 | 工序 | 重点 |
|---|---|---|
| “我想写一本……” | `novel-brainstorm` | 目标、题材、人物、边界 |
| “帮我做大纲/章节规划” | `novel-outline` | 结构模型、章节功能、人物弧线 |
| “写下一章/继续写” | `novel-draft` | 读取记忆、承接上文、完成当前章 |
| “润色/改写这段” | `novel-draft` | 保留叙事功能，改善语言 |
| “审查/看看问题” | `novel-review` | 叙事、人物、语言、canon |
| “同步记忆/更新状态” | `novel-update` | canon、人物、伏笔、进度 |

## 技法速查

| 用户说法或场景 | 推荐技法 | 用法 |
|---|---|---|
| 留白、不要说透、余味 | 不写之写 | 关键情绪或真相不直接解释 |
| 一个场景同时推进多件事 | 一击两鸣 | 情节、关系、信息同步推进 |
| 伏笔、线索、铺垫 | 草蛇灰线 | 前文自然埋入，后文可回收 |
| 含而不露的态度 | 春秋笔法 | 用事实排序表达判断 |
| 画面感、细节、真实 | 白描 | 用动作和具体物件替代抽象形容 |
| 前后对照、命运回响 | 层峦迭翠 | 相似场景产生差异和互文 |
| 表面与内里相反 | 反衬法 | 表面写 A，实际显出 B |
| 跳过重复过程 | 避繁文法 | 用结果、旁人反应或痕迹交代 |
| 一波三折、交锋升级 | 繁章波折法 | 同一场景内多次局势变化 |
| 借旁人说破 | 暗透法 | 让别人说出角色无法直说的东西 |

## 风格匹配

| 作品气质 | 风格文件 |
|---|---|
| 未指定、自然、通用 | `styles/通用中文小说.md` |
| 克制、现实、历史、乡土、战争 | `styles/冷白描.md` |
| 诡异、悬疑、规则、恐怖 | `styles/怪诞悬疑.md` |
| 系统、升级、爽点、强连载 | `styles/系统爽文.md` |

## 推荐格式

```markdown
推荐工序：novel-draft
推荐风格：通用中文小说.md
推荐技法：白描 + 一击两鸣
理由：当前任务需要写清人物行动，同时让场景兼顾情节推进和关系变化。
```
