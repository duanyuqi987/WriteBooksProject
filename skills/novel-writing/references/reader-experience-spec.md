# 读者体验规格

深度融合 iPhone 团队的"体验驱动规格"方法论。不先定技术指标，先定义读者的"人的时刻"。

## 核心理念

> 不要先写"这一章要写什么"。先写"这一章要让读者感受到什么"。

指标可以从糟糕的体验中达成。体验规格从读者的感受出发，再反推技术约束。

## 五步流程

### 第一步：识别时刻

问自己这 5 个问题：
1. 这部小说最重要的一刻是什么？（读者和别人聊起这部小说时会说什么？）
2. 读者在什么时候会放不下？（什么样的场景让他们翻到凌晨 2 点？）
3. 失败时刻最致命的是什么？（读者在哪里弃书？）
4. 理想的情感状态是什么？（读者读完应该有什么感觉？）
5. 读者在现实生活中的什么场景会读这部小说？（通勤？睡前？假期？）

### 第二步：生动描述时刻

把读者的阅读时刻写成故事。让它具体。包含感官细节。

```
时刻模板：

时刻名：[描述性名称]

场景：
- 什么时候：[具体情境，如"深夜 1 点，明天要上班"]
- 在哪里：[环境，如"被窝里，手机屏幕是唯一的光源"]
- 什么人：[读者状态，如"已经很困但还想再看一章"]
- 目标：[读者想要什么，如"知道主角能不能活着走出那个陷阱"]

体验：
- 感受：[主要情绪，如"紧张到不敢呼吸"]
- 节奏：[急促 / 舒缓 / 专注 / 放松]
- 摩擦：[无 / 可接受 / 致命]
- 结果：[读完时的感觉，如"长舒一口气，但马上想知道接下来会发生什么"]
```

### 第三步：提取"感受"

核心感受词（选 2-3 个，不能再多）：

| 感受词 | 含义 | 小说中的体现 |
|--------|------|-------------|
| 放不下 | 每一章结尾都想继续读 | 章末钩子够强 |
| 呼吸感 | 紧张后有喘息的空间 | fallout 章节的节奏 |
| 真实 | 角色像真人，世界可信 | 白描功力、细节密度 |
| 聪明 | 情节设计让读者感觉被尊重 | 伏笔、反转、不套路 |
| 心疼 | 读者对角色产生情感依附 | 角色弧光、内在矛盾 |
| 震撼 | 某些场景的冲击力久久不散 | 高潮场景的战斗力 |
| 温暖 | 残酷世界中的一点点亮色 | 人物关系中的温情时刻 |
| 意外 | 读者没想到但回想起来很合理 | 伏笔兑现、反转 |
| 沉浸 | 读完才发现过了一小时 | 世界观密度、细节丰富度 |

### 第四步：从感受翻译为技术约束

| 感受 | 技术翻译 | 小说中的具体约束 |
|------|---------|-----------------|
| 放不下 | 每章必须有关键悬念未解 | 每章末尾必须检查钩子 |
| 呼吸感 | 高潮后必须有 fallout 章 | 不连续 2 章 climax |
| 真实 | 禁止空洞修饰和 AI 腔 | 每章通过 6 类 AI 检测 |
| 聪明 | 伏笔必须提前埋、自然埋 | 每条伏笔有独立的追踪文件 |
| 心疼 | 角色必须有可感的内在矛盾 | 身份张力不能是套话 |
| 震撼 | 高潮场景的字数投入要够 | 高潮章字数可超目标 20% |
| 温暖 | 不能全程黑暗无亮色 | 每卷至少有 2 处温暖时刻 |
| 意外 | 反转必须有铺垫 | 反转前至少提示 2 次 |
| 沉浸 | 世界观细节密度够 | 每章至少 3 处五感细节 |

### 第五步：定义反模式

这部小说绝对不能发生什么？从读者的角度定义：

```
反模式：
1. 读者跳过一整段不影响理解 → 删掉那段
2. 读者猜到接下来的发展且猜对了，但没有任何意外感 → 太套路
3. 读者分不清两个角色 → 声纹不够差异化
4. 读者看到某段文字觉得"这是 AI 写的" → AI 腔没去干净
5. 读者读完一章觉得"什么都没发生" → 升级感不够
```

## 完整体验规格模板

```markdown
# 体验规格：《书名》

## 人的时刻

### 时刻 1：[名称]
[按上述模板填写]

### 时刻 2：[名称]
[填写]

### 时刻 3：[名称]
[填写]

## 核心感受（最多 3 个）
1. [感受 1]
2. [感受 2]
3. [感受 3]

## 从中推导的技术约束

### 从 [感受 1] 推导：
- [约束 1]
- [约束 2]

## 反模式
这部小说绝对不能：
1. [反模式 1]
2. [反模式 2]

## 成功标准
如何知道体验已经达成：
1. [标准 1]
2. [标准 2]
```

## 验证测试

- **5 追问测试**：对每个约束连问 5 次"为什么"，最终要追溯到人的时刻
- **反指标测试**：哪些指标即使没达成，体验也可能完美？（如"每章字数达标"≠"读起来好"）
- **竞品测试**：如果用不同的技法组合实现了同样的阅读体验，我们能认出来吗？

## 融入写作流程

- **brainstorm 阶段**：定义 2-3 个核心感受和人的时刻
- **outline 阶段**：检查每卷是否服务于体验规格
- **draft 阶段**：每个场景对照感受词自问"读者现在应该感受到什么？"
- **review 阶段**：以读者视角读，实际感受是否匹配目标感受？
