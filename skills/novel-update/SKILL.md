# 章节同步与记忆写入

本工序只做事实同步，不做新的创作判断。它让长篇或持续项目在跨会话写作时保持连续。

## 硬规则

- 只有正文通过 `novel-review` 或用户明确确认定稿后，才执行本工序。
- 不重写正文，不新增剧情判断。
- 重要变化只追加记录，不覆盖历史设定。
- 轻量短篇任务可以不执行完整同步，但要说明未写入记忆。

## 同步清单

### 1. 读取上下文

- 当前章节工作文件
- 纯正文文件
- `project.md`
- `outline.md`
- `memory-snapshot.md`
- `project-state.md`
- 相关人物卡
- 相关伏笔文件

### 2. 更新 canon

检查正文是否产生新事实：

- 新角色、地点、组织、物品
- 关系变化
- 世界规则揭示或修正
- 重要时间线变化
- 角色能力、身份或立场变化

格式：`[chapter-XXX] 变更类型：具体事实`。

### 3. 更新人物

只记录正文中已经发生的变化：

- 欲望变化
- 关系变化
- 信念变化
- 状态变化
- 声纹或行为习惯变化

格式：`[chapter-XXX] 变化内容`。

### 4. 更新伏笔

- 新埋入：创建或更新 `伏笔/foreshadow-xxx.md`
- 中途提示：追加提示记录
- 已兑现：更新为 `paid_off`
- 放弃或改写：记录原因，不静默删除

### 5. 更新大纲与章节状态

- 当前章节 frontmatter 改为 `done`
- 在章节文件写入 2-3 句定稿摘要
- `outline.md` 中对应章节状态改为 `done`
- 如果后续大纲因正文实际走向需要调整，只改未来章节并记录原因

### 6. 更新记忆快照与调度状态

`memory-snapshot.md`：

- 追加本章摘要
- 更新累计字数
- 更新活跃角色、未兑现伏笔、关键 canon
- 保持最近章节摘要简洁

`project-state.md`：

- 当前状态
- 下一章或下一步
- 最近更新时间
- 阻塞点，如无则留空

## 汇报模板

```markdown
## 本轮同步完成

- 章节：
- 字数：
- Canon 变化：
- 角色变化：
- 伏笔变化：
- 大纲状态：
- 下一步：
```

## 红线

- 不把推测写成 canon。
- 不静默删除伏笔、角色或历史记录。
- 不让 `project-state.md` 覆盖章节 frontmatter 的真实状态。
- 不为了同步而改正文内容。
